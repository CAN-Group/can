URL parameters:
coords: GPS positions of subsequent points from start to finish; longitude (float) and latitude (float) of a point separated by ","; subsequent points separated by "|"; eg. coords=1.0,2.0|3.0,4.0
counties: optional IDs (strings) of counties to avoid; IDs separated by ","; eg. counties=t0201,t3203
profile: optional name (string) of routing profile; see /static/routing_profiles.json for a list of available profiles; default profile: profile=car-fast
variant: optional route variant - integer in range <0,4>; default variant: 0; eg. variant=1
format: output path file format - string; accepted values: gpx, kml, geojson; default format: gpx; eg. format=kml

example: /api/v1/route?coords=16.298218,54.239551|17.002716,54.457266&counties=t3213&profile=car-eco
